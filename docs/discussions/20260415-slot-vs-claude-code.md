## 核心问题

> Sloth Agent 用强制流程开发项目，和 Claude Code 比有什么不同？会比 Claude Code 做得好吗？

| 简单 | 无需配置，clone 就能用 |

## 灵魂拷问

> Claude Code 用起来也挺顺的，为什么要强制 TDD？为什么要先写设计再写代码？

很多开发者会觉得：
- "我想快点开始写代码"
- "我只是改个小 bug，需要走流程吗？"
- "设计文档写完就过时了"

---

## Claude Code 可借鉴的功能

### 1. 项目上下文扫描

Claude Code 启动时自动扫描项目结构，识别语言、框架、依赖、git 状态。

**借鉴**：Sloth Agent 启动时做 `sloth scan`，建立项目画像。

### 2. 轻量级 Approval（不打断流）

```
Claude Code:
  - 低风险操作 → 直接执行
  - 中风险操作 → 简短提示（y/n）
  - 高风险操作 → 暂停等待确认
  - 连续同意 3 次 → 后续同类操作自动放行
```

**借鉴**：Approval 分级，不是一刀切的"审批"。

### 3. 增量验证（小循环）

```
Claude Code:
  1. 写代码 → 2. 运行测试 → 3. 看看报错 → 4. 修复 → 5. 再运行
  （快速反馈 loop）
```

**借鉴**：TDD 是每个微任务做小循环，不是"写完所有测试再写代码"。

### 4. 错误恢复机制

报错后尝试修复，验证是否修复，如果失败换个思路，不重复失败的修复。

### 5. 上下文压缩

压缩对话历史，保留最近修改、关键错误、当前状态，丢弃已解决的讨论。

### 6. 可见性（不闷头干活）

执行命令前告诉你要做什么，显示命令输出，展示 diff，失败时说明原因。

## 关键结论

| Claude Code 做法 | Sloth Agent 应该学 | Sloth Agent 不应该学 |
|-----------------|-------------------|---------------------|
| 直接执行 | 快速反馈循环 | 强制 7 步流程 |
| 按需了解项目 | 启动时扫描 | 过度的前期设计 |
| 灵活的 approval | 分级 approval | 僵化的审批 |
| 上下文压缩 | 记忆遗忘 | 记住一切 |

---

## 建议方向

**核心价值 = Claude Code 的效率 + 一点点结构化**，而不是"严格的流程"。

```
Sloth Agent 风格：
1. 启动 → 扫描项目，告诉你"我理解你的项目是 XX，用 XX 技术栈"
2. 任务 → "我要做 XX，会修改 YY 文件，需要你的批准吗？(y/n/a=全部同意)"
3. 执行 → 显示命令和输出
4. 验证 → "测试通过了，变更已提交"
5. 结束 → "完成，耗时 XX 分钟"
```
