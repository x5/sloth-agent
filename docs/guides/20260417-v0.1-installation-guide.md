## 5. 配置 LLM Provider

Sloth Agent 需要至少一个 LLM Provider 的 API Key 才能工作。v0.1 支持 DeepSeek 和 Qwen。

### 5.1 环境变量方式

在项目根目录创建 `.env` 文件：

```bash
# DeepSeek API（编码/调试主力）
DEEPSEEK_API_KEY=your-deepseek-api-key

# Qwen API（代码审查）
QWEN_API_KEY=your-qwen-api-key
```

### 5.2 配置文件方式

复制默认配置：

```bash
cp configs/agent.yaml.example configs/agent.yaml
```

编辑 `configs/agent.yaml`，填入 API Key：

```yaml
llm:
  providers:
    deepseek:
      api_key: "${DEEPSEEK_API_KEY}"
      base_url: "https://api.deepseek.com/v1"
      models:
        coding: "deepseek-v3.2"
        reasoning: "deepseek-r1-0528"
    qwen:
      api_key: "${QWEN_API_KEY}"
      base_url: "https://dashscope.aliyuncs.com/compatible-mode/v1"
      models:
        review: "qwen3.6-plus"
```

> **注意**：v0.1 阶段只需要配置 2 个 Provider（DeepSeek + Qwen）即可跑通完整流水线。

## 8. 创建你的第一个 Plan

Plan 是 Sloth Agent v0.1 的输入。创建一个简单的 Plan 文件用于测试：

```markdown
# My First Plan

## Goal
Create a simple CRUD API for a todo list app using Python FastAPI.

## Tasks

### Task 1: Project Setup
- Create `main.py` with FastAPI app
- Create `models.py` with Todo model (id, title, completed)
- Create `requirements.txt` with dependencies

### Task 2: API Endpoints
- GET /todos - list all todos
- POST /todos - create a todo
- GET /todos/{id} - get a todo
- PUT /todos/{id} - update a todo
- DELETE /todos/{id} - delete a todo

### Task 3: Tests
- Write unit tests for all endpoints
- Achieve 80%+ coverage

## Constraints
- Use in-memory storage (no database)
- Follow REST conventions
- All tests must pass before completion
```

## 9. 运行自主流水线

```bash
# 运行完整流水线
uv run sloth run --plan path/to/your-plan.md

# 查看状态（另一个终端）
uv run sloth status

# 查看日志
uv run sloth logs --level INFO --limit 50
```

### 流水线执行流程

```
Plan ─→ [Builder Agent] ─→ Gate 1 ─→ [Reviewer Agent] ─→ Gate 2 ─→ [Deployer Agent] ─→ Gate 3 ─→ Done
         解析+编码+测试       lint        代码审查+验证      test       部署+验证           smoke
         deepseek            type        qwen              coverage   deepseek            test
```

### 门控说明

| 门控 | 位置 | 规则 | 失败处理 |
|------|------|------|---------|
| **Gate 1** | Builder → Reviewer | lint 通过 + type check 通过 + 测试通过 | Builder 自动修复，最多重试 3 次 |
| **Gate 2** | Reviewer → Deployer | blocking_issues 为空 + coverage ≥ 80% | 打回 Builder 修复 |
| **Gate 3** | Deployer → Done | smoke test 通过 | 自动回滚 + 通知 |

## 10. 常见问题

### Q: 测试失败怎么办？

```bash
# 查看具体失败的测试
uv run pytest tests/ -v --tb=long

# 只运行某个测试文件
uv run pytest tests/core/test_runner.py -v
```

### Q: 只有一个 Provider 的 API Key 能跑吗？

可以。在 `configs/agent.yaml` 中只配置一个 Provider，将 Reviewer 和 Builder 指向同一个 Provider 的不同模型即可。但 Reviewer 的审查效果会打折扣（同一模型审自己代码几乎无价值）。

### Q: 运行时卡住了怎么办？

1. 检查网络连接是否正常
2. 确认 API Key 有效且额度充足
3. 查看日志: `uv run sloth logs --level ERROR`
4. 检查 Git 状态: `git status`
5. 如需回滚: `uv run python -c "from sloth_agent.reliability.checkpoint import CheckpointManager; cm = CheckpointManager(); cm.rollback_to('sloth/stage/test-stage')"`

### Q: 如何查看当前 v0.1 已实现的功能？

