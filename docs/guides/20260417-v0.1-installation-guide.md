## 1. 系统要求

| 要求 | 说明 |
|------|------|
| **操作系统** | macOS 12+ / Windows 11 (WSL2 或 PowerShell) / Linux |
| **Python** | 3.11+（通过 uv 自动管理） |
| **uv** | 0.5+（Python 包管理器） |
| **Git** | 2.30+（需要 Git tag 功能） |
| **网络** | 需访问 DeepSeek / Qwen API |

---

## 2. 一键安装

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/x5/sloth-agent/master/scripts/install.sh | bash
```

### Windows (PowerShell)

```powershell
iwr -useb https://raw.githubusercontent.com/x5/sloth-agent/master/scripts/install.ps1 | iex
```

安装脚本会自动完成：克隆仓库、安装 uv、创建虚拟环境、安装依赖、初始化目录结构。

---

## 3. 手动安装（可选）

如果一键安装失败或需要自定义：

```bash
# 克隆项目
git clone https://github.com/x5/sloth-agent.git
cd sloth-agent

# 安装依赖
uv sync --dev

# 验证 CLI
uv run sloth --help
```

应输出 CLI 帮助信息，包含 `run`、`chat`、`status`、`config`、`init` 命令。

## 5. 配置层级说明

Sloth Agent 的配置有三个层级，优先级从高到低：

| 优先级 | 层级 | 位置 | 说明 |
|--------|------|------|------|
| 1（最高） | local | `[项目目录]/config.json` | 仅当前项目生效 |
| 2 | project | 当前项目的配置 | 项目级覆盖 |
| 3（最低） | user | `~/.sloth-agent/config.json` | 所有项目共用 |

API Key 的解析顺序类似：

| 优先级 | 来源 | 路径 | 说明 |
|--------|------|------|------|
| 1（最高） | 项目 .env | `[项目目录]/.env` | 仅当前项目生效 |
| 2 | 全局 .env | `~/.sloth-agent/.env` | 所有项目共用，项目 .env 可覆盖 |
| 3 | 系统环境变量 | 当前 shell 的 `env` | 仅当前 shell session 生效 |

### 查看当前配置

```bash
sloth config show              # 显示合并后的完整配置
sloth config show --scope user # 仅显示用户级配置
sloth config env               # 检查 API Key 配置状态
sloth config validate          # 验证配置是否正确
```

### 支持的 Provider

| Provider | 默认模型映射 | 用途 |
|----------|-------------|------|
| **DeepSeek** | coding: deepseek-v3.2<br>reasoning: deepseek-r1-0528 | 主力编码与推理 |
| **Qwen** | review: qwen3.6-plus | 代码审查 + 低成本推理 |
| **Kimi** | vision: kimi-k2.5 | 视觉编码（可选） |
| **GLM** | coding: glm-5.1 | 编程（可选） |
| **MiniMax** | evolution: minimax-m2.7 | 自进化（可选） |
| **Xiaomi** | coding: mimo-v2-pro | 高强度 Agent（可选） |

> **注意**：v0.2.0 阶段至少需要配置 1 个 Provider 即可工作，推荐配置 DeepSeek + Qwen 两个来跑通完整流水线。

## 8. 创建你的第一个 Plan

Plan 是 Sloth Agent v0.2.0 的输入。创建一个简单的 Plan 文件用于测试：

```markdown
# My First Plan

## Goal
Create a simple CRUD API for a todo list app using Python FastAPI.

## Tasks

### Task 1: Project Setup
- Create `main.py` with FastAPI app
- Create `models.py` with Todo model (id, title, completed)
- Create `requirements.txt` with dependencies

### Task 2: API Endpoints
- GET /todos - list all todos
- POST /todos - create a todo
- GET /todos/{id} - get a todo
- PUT /todos/{id} - update a todo
- DELETE /todos/{id} - delete a todo

### Task 3: Tests
- Write unit tests for all endpoints
- Achieve 80%+ coverage

## Constraints
- Use in-memory storage (no database)
- Follow REST conventions
- All tests must pass before completion
```

## 9. 运行自主流水线

```bash
# 运行完整流水线
uv run sloth run --plan path/to/your-plan.md

# 查看状态（另一个终端）
uv run sloth status

# 查看日志
uv run sloth logs --level INFO --limit 50
```

### 流水线执行流程

```
Plan ─→ [Builder Agent] ─→ Gate 1 ─→ [Reviewer Agent] ─→ Gate 2 ─→ [Deployer Agent] ─→ Gate 3 ─→ Done
         解析+编码+测试       lint        代码审查+验证      test       部署+验证           smoke
         deepseek            type        qwen              coverage   deepseek            test
```

### 门控说明

| 门控 | 位置 | 规则 | 失败处理 |
|------|------|------|---------|
| **Gate 1** | Builder → Reviewer | lint 通过 + type check 通过 + 测试通过 | Builder 自动修复，最多重试 3 次 |
| **Gate 2** | Reviewer → Deployer | blocking_issues 为空 + coverage ≥ 80% | 打回 Builder 修复 |
| **Gate 3** | Deployer → Done | smoke test 通过 | 自动回滚 + 通知 |

## 11. 常见问题

### Q: 测试失败怎么办？

```bash
# 查看具体失败的测试
uv run pytest tests/ -v --tb=long

# 只运行某个测试文件
uv run pytest tests/core/test_runner.py -v
```

### Q: 只有一个 Provider 的 API Key 能跑吗？

可以。在配置中只设置一个 Provider，将 Reviewer 和 Builder 指向同一个 Provider 的不同模型即可。但 Reviewer 的审查效果会打折扣（同一模型审自己代码几乎无价值）。

### Q: 运行时卡住了怎么办？

1. 检查网络连接是否正常
2. 确认 API Key 有效且额度充足
3. 查看日志: `uv run sloth logs --level ERROR`
4. 检查 Git 状态: `git status`
5. 检查配置: `sloth config validate`
6. 如需回滚: `uv run python -c "from sloth_agent.reliability.checkpoint import CheckpointManager; cm = CheckpointManager(); cm.rollback_to('sloth/stage/test-stage')"`

### Q: 如何查看当前 v0.2.0 已实现的功能？

```bash
uv run python -c "
```
